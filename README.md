# Quest Game

Quest Game - это консольная текстовая игра-квест на C++. Игрок попадает в
подземный лабиринт, перемещается между локациями, решает головоломки,
открывает сундуки, использует зелья лечения и должен найти выход.

Игра изначально планировалась на русском языке, но из-за проблем с кодировкой
в консоли основной игровой текст оставлен на английском.

## Возможности

- 4 связанные локации;
- ASCII-отрисовка каждой локации;
- закрытые локации, которые открываются после решения головоломок;
- разные типы головоломок;
- урон за неправильные ответы;
- здоровье игрока;
- инвентарь;
- 2 типа зелий лечения;
- сундуки с предметами;
- сохранение и загрузка игры;
- условие победы в финальной локации.

## Структура проекта

Проект разделён на несколько классов. Каждый класс находится в отдельной паре
файлов `.h` и `.cpp`.

- `Game` - основная логика игры, меню, команды, сохранение и загрузка;
- `Player` - здоровье игрока, текущая локация и инвентарь;
- `Location` - данные локации, описание, связи, ASCII-карта и сундуки;
- `Potion` - данные о зельях лечения;
- `Chest` - состояние сундука и список предметов внутри.

Основные файлы проекта:

- `main.cpp` - точка входа в программу;
- `Makefile` - файл для сборки проекта;
- `README.md` - описание проекта.

## Сборка

Для сборки используется `Makefile` и компилятор `g++`.

Собрать проект:

mingw32-make all

Очистить файлы сборки:

mingw32-make clean

## Запуск

.\quest_game.exe

## Управление

В главном меню:

- `1` - начать новую игру;
- `2` - загрузить сохранение;
- `0` - выйти.

Во время игры:

- `1` - открыть сундук;
- `2` - показать инвентарь;
- `3` - перейти в другую локацию;
- `4` - выпить зелье;
- `5` - сохранить игру;
- `0` - выйти из игры.

## Зелья

При неверном решении головоломки игрок получает урон.
Его можно восполнить за счет зелья. Что бы выпить зелье нужно нажать 4 во время игры и ввести ID зелья, которое хотите выпить

## Локации

Всего в игре 4 локации.
Для перехода между локациями игрок должен решать головоломки. Переход осуществляется по id локации.
Локации связаны последовательно - из "1" можно перейти только во "2", из "2" в "1" и "3" и так далее.

## Подсказка (легенда)

Легенда выводится при каждом ходе в консоли на английском языке
Legend: @ - player, C - chest, > - exit
Русский вариант:
@ - Игрок, C - сундук, > - выход из локации

## Разделители

"-----------" Визуально отделяют блоки внутри одного хода

"===========" Разделение ходов игрока в терминале

## Сохранение

Игра сохраняет прогресс в файл `save.txt`. В сохранении хранятся:

- текущая локация игрока;
- здоровье игрока;
- инвентарь;
- открытые локации;
- открытые сундуки.

Файл `save.txt` не добавляется в Git, потому что это локальный файл сохранения
игрока
